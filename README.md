# Linux/mac에서의 컴파일 및 실행

1. 터미널을 실행.

2. 'cd'를 통해 이 프로젝트 파일로 이동합니다.

3. 'gcc nuguri.c -o nuguri'를 입력하여 컴파일을 진행합니다.

4. 컴파일이 완료 된 뒤 './nuguri'를 입력하여 게임을 실행하면 됩니다.


# Windows에서의 컴파일

**(Linux/mac에서의 컴파일과 동일한 환경을 구성하기 위해 gcc 컴파일러의 사용을 권장합니다.)**

**(MinGW-w64 등)**



1. cmd 또는 powershell을 실행합니다.

2. 'cd' 명령어를 통해 이 폴더로 이동해옵니다.

3. 'gcc nuguri.c -o nuguri.exe'을 입력하여 컴파일합니다.

4. 컴파일이 완료되면 'nuguri.exe'를 입력하여 게임을 실행하면 됩니다.

# 문제 해결 과정

* 플레이어 캐릭터의 낙하 속도가 1프레임 당 2이상이 되는 경우 낙하 방향에 존재하는 바닥을 무시하고 통과해버리는 현상이 존재했습니다. 이에 따라 낙하 속도를 1프레임 당 1이 되도록 이동 속도를 1로 제한했습니다.

* 윈도우 환경 호환을 위해 clrscr()을 사용했습니다. 그러자 윈도우에서 구동은 되나 매 프레임 화면이 리프레시되어 화면이 점멸하는 증상이 발생(화면을 매 프레임 전부 지웠다 재작성하기 때문입니다), 이를 해결하기 위해 분기하여 커서 위치를 0, 0으로 이동하여 한줄씩 오브젝트들을 그려나가되, 그 커서가 그려나가는 것을 플레이어가 인지하지 않도록 그 커서를 숨겼습니다.
그 후 분기 처리를 통해 각 os에서 정상적인 구동이 가능할 수 있도록 작업했습니다.

* 동적 맵 할당의 경우 [맵의 가로 크기][맵의 세로 크기][각 스테이지]로 삼중 처리 하여 하나씩 읽어나가면 될 것 같다는 생각은 듭니다만, 저희 조에게 주어진 시간과 역량을 고려했을때, 상기 버그와 에러 처리에 집중하는 것이 좋을 것 같다로 의견이 모아져 그러한 작업에 집중했습니다.
